问题一：user = 40
问题二：user2 = 40

问题三：user3 = 20

方法的重写：
1.存在父子类关系，（子类重写父类的方法）
2.如果子类和父类都有相同的方法，那么遵循动态绑定机制

动态绑定机制：再程序的执行过程中，如果调用了对象的成员方法，那么会将对象的方法和对象实际的内存绑定。与属性无关 也就是说。

Java05_overwrite再调用sum方法时，会先去寻找User中的sum方法，没有再去寻找Person中的方法。但是对于其中的属性，就不遵循动态绑定机制。也就是那个方法调用，就用那个类中的属性。
